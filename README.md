# Prospective Customer Prediction

Predicts whether a bank customer will subscribe to a term deposit based on demographic, behavioral, and campaign interaction data. Built for a marketing campaign optimization use case — helping the bank target the right customers and maximize conversion.

**Best model:** Random Forest (tuned) — **91% accuracy, AUC 0.73**

---

## Problem Statement

A leading bank wants to promote a newly launched product via a marketing campaign. To maximize ROI, the campaign should focus on customers most likely to subscribe. This project builds a binary classifier to predict term deposit subscription (`yes` / `no`) based on customer data.

---

## Dataset

**Source:** Bank marketing dataset (UCI-style, Portuguese bank telemarketing campaigns)

**File:** `data/bank_marketing.csv`

**Shape:** ~41,000 rows × 21 columns

| Column Group | Features |
|---|---|
| Client demographics | age, job, marital, education, default, housing, loan |
| Last campaign contact | contact, month, day_of_week, duration |
| Campaign history | campaign, pdays, previous, poutcome |
| Economic indicators | employment_variation_rate, consumer_price_index, consumer_confidence_index, 3_months_euribor_rate, number_of_employees_in_the_bank |
| **Target** | **term_deposit** (yes / no) |

---

## How to Run

### Step 1 — Install

```bash
pip3 install -r requirements.txt
```

### Step 2 — Add data

Place `bank_marketing.csv` in `data/`.

### Step 3 — Train all models

```bash
python3 src/train.py
```

Train specific model only:
```bash
python3 src/train.py --model rf     # Random Forest (with RandomizedSearchCV)
python3 src/train.py --model lr     # Logistic Regression (with GridSearchCV)
python3 src/train.py --model rf --no-tune  # Skip tuning, use preset params
```

Outputs:
- `reports/baseline_comparison.csv` — accuracy across all 5 baseline models
- `reports/figures/roc_curve_rf.png` — ROC curve for Random Forest
- `models/random_forest.pkl` — saved best model
- `models/logistic_regression.pkl`
- `models/scaler.pkl`

### Step 4 — Predict on new data

```bash
python3 src/predict.py --input data/new_customers.csv
python3 src/predict.py --input data/new_customers.csv --model lr
```

Output: `reports/predictions.csv` with `predicted_subscription` (0/1) and `probability` columns.

### Step 5 — (Optional) Explore in notebook

```bash
jupyter lab notebooks/
```

## Preprocessing Pipeline

| Step | Detail |
|---|---|
| Column standardisation | Lowercase, replace spaces with `_` |
| Education consolidation | `basic.4y`, `basic.6y`, `basic.9y` → `basic.school` |
| `pdays` engineering | 999 (not contacted) → `prev_c=no`; others → `prev_c=yes`; drop `pdays` |
| Deduplication | Drop exact duplicate rows |
| Outlier capping (IQR) | `age` cap 69.5, `duration` cap 644.5, `campaign` cap 6.0 |
| Encoding | month/day → integer; default/housing/loan → {1,0,-1}; job/marital/education/contact/poutcome → one-hot |
| Feature selection | Mutual information → drop 22 low-importance features |
| Scaling | StandardScaler on 9 continuous numeric columns only |
| Split | 80/20 train/test, `random_state=42` |

## Hyperparameter Tuning

**Random Forest** — RandomizedSearchCV (100 iterations, 5-fold CV):
```python
n_estimators: [5, 20, 50, 100]
max_features: ['sqrt', 'log2']
max_depth: linspace(10, 120, 12)
min_samples_split: [2, 6, 10]
min_samples_leaf: [1, 3, 4]
bootstrap: [True, False]
```
Best params stored in `src/config.RF_PARAMS`.

**Logistic Regression** — GridSearchCV:
```python
C: logspace(-4, 4, 50)
penalty: ['l1', 'l2']
```
Best params stored in `src/config.LR_PARAMS`.
